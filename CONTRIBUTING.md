# Contributing

## Working Style

Use a professional workflow:

1. Create or select an issue.
2. Work in a separate branch.
3. Keep changes focused.
4. Test before opening a pull request.
5. Explain what changed and why.

## Branch Names

Use clear branch names:

- `feature/webview-shell`
- `fix/android-build`
- `docs/release-notes`
- `release/1.0.0`

## Commit Style

Use short conventional commit messages:

- `feat: add app icon`
- `fix: correct WebView settings`
- `docs: update build instructions`
- `chore: clean project files`

## Pull Request Checklist

- [ ] The change has a clear purpose.
- [ ] The Android project still builds.
- [ ] WebView behavior was tested when relevant.
- [ ] No signing keys or secrets were committed.
- [ ] README or release notes were updated when behavior changed.
